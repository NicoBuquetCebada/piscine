# <img width="27" heigth="27" src="https://raw.githubusercontent.com/kube/vscode-42header/master/42.png"> Piscine
Este repositorio contiene los proyectos desarrollados durante la piscina de 42, el proceso intensivo de selección del llamado [cursus](https://github.com/NicoBuquetCebada/cursus).

La piscina constituye una experiencia de aprendizaje altamente exigente en la que los participantes deben resolver problemas de programación en lenguaje C mediante aprendizaje autónomo, colaboración y desarrollo práctico de proyectos.

## ¿Que es la piscina?
La piscina es un programa intensivo de aproximadamente cuatro semanas que forma parte del proceso de admisión.

Durante este periodo los participantes deben enfrentarse a:

- Ejercicios diarios de programación en C
- Proyectos progresivamente más complejos
- Evaluaciones entre estudiantes (peer evaluation)
- Trabajo colaborativo bajo presión
- El objetivo es evaluar no solo los conocimientos técnicos, sino también la capacidad de aprendizaje autónomo, la resiliencia y la habilidad para trabajar en equipo.

## Índice de proyectos
Este repositorio está organizado según las principales etapas de la piscina:

| Proyecto | Descripción |
|--------|-------------|
| **shell** | Primeros proyectos centrados en el uso del shell de Linux y en la comprensión de conceptos básicos del sistema. |
| **c** | Proyectos en lenguaje C destinados a construir una base sólida en programación estructurada, manipulación de memoria y lógica algorítmica. |
| **rush** | Proyectos grupales realizados durante los fines de semana. Están diseñados para fomentar la colaboración y la resolución rápida de problemas bajo plazos muy ajustados. |
| **bsq** | Proyecto final de la piscina, desarrollado en parejas, cuyo objetivo es aplicar de forma integrada los conocimientos adquiridos durante el programa. |

# <img width="27" heigth="27" src="https://raw.githubusercontent.com/kube/vscode-42header/master/42.png"> Piscine
This repository contains the projects developed during the 42 piscine, the intensive selection process for the so-called [cursus](https://github.com/NicoBuquetCebada/cursus).

The piscine is a highly demanding learning experience in which participants must solve programming problems in the C language through autonomous learning, collaboration, and hands-on project development.

## What is the piscine?
The piscine is an intensive program lasting approximately four weeks and forms part of the admission process.

During this period participants are required to face:

- Daily programming exercises in C  
- Progressively more complex projects  
- Peer evaluations between students (peer evaluation)  
- Collaborative work under pressure  

The objective is to assess not only technical knowledge, but also the capacity for autonomous learning, resilience, and the ability to work effectively as part of a team.

## Project index
This repository is organized according to the main stages of the piscine:

| Project | Description |
|--------|-------------|
| **shell/** | First projects focused on the use of the Linux shell and the understanding of basic system concepts. |
| **c/** | Projects written in C aimed at building a solid foundation in structured programming, memory manipulation, and algorithmic logic. |
| **rush/** | Group projects carried out during weekends. They are designed to foster collaboration and rapid problem-solving under very tight deadlines. |
| **bsq/** | Final project of the piscine, developed in pairs, whose objective is to apply in an integrated way the knowledge acquired during the program. |
